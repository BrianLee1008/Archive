이 skill은 TIL 본문에 들어갈 짧은 코드 예제를 만드는 진입점이다.

## 입력 처리

- 인자로 요청 내용을 받는다(예: `/snippet B+Tree 노드 분할 의사코드 (Python)`).
- 모호하면 다음을 한 번에 묻고 진행:
  - 언어(예: Python, TypeScript, Go, …)
  - 런타임/버전(예: Node 20, Python 3.12)
  - 보여주려는 핵심 동작 한 줄

## 실행 절차

1. **til-snippet-writer agent를 호출**한다. 요청·언어·버전·핵심 동작을 함께 전달.
2. agent가 반환한 코드(검증 결과 포함)를 사용자에게 보여준다.
3. 사용자에게 다음을 묻는다:
   - "이 스니펫을 어떤 파일에 삽입할까요? (또는 그냥 보고만 끝낼까요?)"
   - 파일 경로 + 삽입 위치(예: `archive/cs/B-트리와-B+트리.md` 의 `## 예시` 섹션 뒤)를 받으면 Edit으로 삽입.
   - 사용자가 추가 변형(예: "주석 더 줄여", "TypeScript로 바꿔")을 요청하면 agent를 다시 호출.

## 출력 형식

agent 응답(코드 + 검증 + 기대 출력 + 주의)을 그대로 보여준 뒤, 마지막에:

> 어디에 넣을까요? 파일 경로와 삽입 위치를 알려주세요. 그냥 보고 끝나면 `done`.

## 주의

- 실행 검증이 불가했으면(런타임 미설치 등) 그 사실을 숨기지 않는다.
- 본인이 직접 만든 임시 디렉토리·테스트 파일은 작업 후 정리한다.
- 너무 긴 예제(50줄 이상)는 분할 권장(한 글에 짧은 스니펫 여러 개가 더 읽기 좋음).
