이 skill은 새 TIL 글을 쓰기 전 사전 리서치를 수행하는 진입점이다.

## 입력 처리

- 인자가 있으면 그것이 주제다(예: `/research React 19 use훅`).
- 인자가 비어있으면 사용자에게 "어떤 주제를 리서치할까요?"라고 한 번 묻는다.
- 주제가 너무 광범위하면(예: "React") 한 번 더 좁힌다("React의 어떤 부분? 컴포넌트 모델, 상태관리, SSR 중 하나로 좁혀주세요").

## 실행 절차

1. **til-researcher agent를 호출**해서 주제를 넘긴다. 이때 agent에는 다음을 명시:
   - 한국어 응답
   - 공식문서 1순위 출처 정책 준수
   - 출력 포맷(개요/핵심개념/Fact/관점/구조/카테고리/관련글) 준수
2. agent가 반환한 리서치 노트를 사용자에게 그대로 보여준다.
3. 사용자에게 다음 액션을 제안: "이 내용으로 글 초안을 만들까요? (`archive/<카테고리>/<파일명>.md`)"
   - 사용자가 승인하면:
     - frontmatter(title/date/tags/sources)를 채운 빈 골격을 Write로 생성
     - References 섹션에 agent가 정리한 출처 URL을 모두 옮겨둔다
   - 거절하면 추가 리서치 요청을 받거나 그대로 종료.

## 출력 형식

agent 응답을 그대로 보여주되, 마지막에 다음 한 줄을 추가:

> 다음에 할 일: 초안을 만들까요(`예`) / 다른 각도로 더 리서치(`자유 입력`) / 그만(`아니오`)?

## 주의

- 리서치 단계에서는 `archive/` 안에 파일을 만들지 않는다(사용자가 명시적으로 초안 생성을 승인할 때만).
- 출처 URL은 사용자에게 보여주는 답변뿐 아니라 추후 frontmatter `sources`에 들어갈 수 있도록 보존한다.
- 이미 같은 주제 글이 있다고 agent가 보고하면, 새로 만들지 말지를 사용자에게 묻는다.
