이 skill은 주어진 주제를 **"1 What + 5 Whys"** 구조의 완성된 TIL 초안으로 작성하는 진입점이다.

## 입력 처리

- **주제 인자**: 있으면 그것이 주제다 (예: `/write React 19 use훅`).
- **주제 없음**: 사용자에게 "어떤 주제로 글을 쓸까요?"라고 묻는다.
- **주제가 너무 광범위**: 한 번 더 좁혀준다 (예: "React"가 아니라 "React의 어떤 부분?").

## 선택사항: 리서치 노트 통합

사용자가 이미 `/research <주제>`로 리서치를 완료한 경우:
- 리서치 노트의 경로를 선택적으로 받는다.
- 없으면, "이전에 `/research`로 리서치 노트를 만들었나요? (경로를 알려주시면 참고할게요)"라고 제안.

## 실행 절차

1. **til-writer agent 호출**: 주제 + 리서치 노트 경로(있으면)를 전달.
   - 에이전트에는 다음을 명시:
     - 한국어 응답
     - "1 What + 5 Whys" 6-섹션 구조 준수
     - CLAUDE.md 컨벤션(frontmatter, 파일명, 톤, 출처 정책) 준수
     - 중복 파일 확인 후 보고

2. **agent가 생성한 파일 보고**: 생성된 경로와 섹션 목록을 사용자에게 보여준다.
   ```
   ✓ /Users/ingjobs/archive/자료/Kotlin-Collection.md 생성 완료
   
   섹션:
   - 정의
   - 왜 등장했나
   - 어떻게 동작하나
   - 장단점
   - 무엇과 다른가
   - 언제 쓸까
   - References
   ```

3. **다음 액션 제안**: 3가지 옵션 제시:
   - "이 글을 리뷰할까요? (`/review /Users/ingjobs/archive/자료/Kotlin-Collection.md`)"
   - "코드 예제를 더 추가하고 싶으면 `/snippet` 사용"
   - "사실관계 검증이 필요하면 `/factcheck /Users/ingjobs/archive/자료/Kotlin-Collection.md`"

## 출력 형식

agent가 생성한 파일의 경로와 간단한 성공 메시지 + 다음 액션.

## 주의

- 생성 단계에서 리서치가 완전하지 않을 수 있으므로, 사실 검증은 추후 `/factcheck`에서.
- 중복 파일이 있으면 새로 덮어쓰지 말고, "이미 같은 주제 파일이 있습니다. 수정할까요, 새로 만들까요?"라고 묻기.
- 코드가 필요하면 본문에 간단한 예제만 넣고, 상세하고 정교한 스니펫은 `/snippet` 권장.
